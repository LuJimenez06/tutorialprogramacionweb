Formularios

Los formularios en HTML son una de las herramientas más importantes para interactuar con los usuarios en una página web. Permiten a los usuarios enviar datos al servidor, ya sea para autenticación, búsqueda, encuestas, o cualquier otra forma de entrada de información. A continuación, te presento una guía sobre los formularios en HTML.


Estructura Básica de un Formulario
La estructura básica de un formulario en HTML se define utilizando la etiqueta <form>. Dentro de esta etiqueta, se colocan diferentes elementos de entrada como campos de texto, botones de envío, casillas de verificación, entre otros.
La etiqueta <form> es el contenedor principal para todos los elementos de un formulario.
Atributos comunes:
  • action: Define la URL a la que se enviarán los datos del formulario.
  • method: Especifica el método HTTP que se utilizará para enviar los datos (GET o POST).
  • name: Especifica el nombre del formulario
  • target: Especifica dónde mostrar la respuesta que se recibe después de enviarla La forma
  • enctype: Define el tipo de codificación de datos para el formulario (importante para la carga de archivos).
  • autocomplete: Especifica si un formulario debe tener la función de autocompletar activada o desactivada
  • novalidate: Especifica que el formulario no debe validarse cuando se envía
  • accept-charset: Especifica las codificaciones de caracteres utilizadas para el envío de formularios
  • rel: Especifica la relación entre un recurso vinculado y el recurso actual documento


<form action="/submit" method="post">

... (Elementos del formulario) ...

</form>



Elementos Comunes del Formulario

<input>
La etiqueta <input> es versátil y se utiliza para una variedad de entradas de datos.
  • type: Define el tipo de dato que el campo acepta, como texto, contraseña, correo electrónico, número, etc.
    1. button: Crea un botón simple que no tiene funcionalidad predefinida. Para hacer que el botón realice alguna acción, se debe usar JavaScript.

    2. checkbox: Crea una casilla de verificación que el usuario puede marcar o desmarcar. Se utiliza cuando el usuario tiene la opción de seleccionar una o más opciones de un conjunto.

    3. color: Muestra un selector de color que permite al usuario elegir un color.

    4. date: Muestra un selector de fecha que permite al usuario seleccionar una fecha (año, mes, día).

    5. datetime-local: Permite al usuario seleccionar una fecha y hora sin zona horaria (año, mes, día, hora y minutos).

    6. email: Campo de texto diseñado para ingresar direcciones de correo electrónico. Incluye validación automática para asegurar que el valor ingresado tiene el formato correcto de un correo electrónico.

    7. file: Crea un botón que permite al usuario seleccionar un archivo para cargar.

    8. hidden: Campo oculto que no se muestra en la página pero envía un valor especificado al servidor cuando se envía el formulario.

    9. image: Muestra una imagen que, al hacer clic, envía el formulario. Es similar a un botón de envío (submit).

    10. month: Permite al usuario seleccionar un mes y un año, pero no un día.

    11. number: Campo de entrada que solo acepta números. Permite definir un rango de valores y añadir controles para aumentar o disminuir el número.

    12. password: Campo de texto que oculta el texto que se ingresa, para proteger información sensible como contraseñas.

    13. radio: Crea un botón de opción. Los botones de opción permiten al usuario seleccionar solo una opción de un conjunto de opciones.

    14. range: Crea un control deslizante que permite al usuario seleccionar un valor dentro de un rango definido.

    15. reset: Crea un botón que restablece todos los campos del formulario a sus valores iniciales.

    16. search: Campo de texto diseñado para ingresar términos de búsqueda. Similar a text, pero con algunas optimizaciones de estilo para interfaces de búsqueda.

    17. submit: Crea un botón que envía el formulario cuando se hace clic en él.

    18. tel: Campo de texto diseñado para ingresar números de teléfono. No realiza validación automática del formato, pero se espera que el usuario ingrese un número de teléfono válido.

    19. text: Campo de texto estándar para ingresar datos de texto.

    20. time: Permite al usuario seleccionar una hora (horas y minutos).

    21. url: Campo de texto diseñado para ingresar URLs. Incluye validación automática para asegurarse de que el valor ingresado es una URL válida.

    22. week: Permite al usuario seleccionar una semana y un año.

  • name: Asigna un nombre al campo para identificar los datos cuando se envían.
  • value: Define el valor inicial del campo.


<textarea>
Se utiliza para permitir la entrada de texto en varias líneas.
  • rows: Número de filas visibles.
  • cols: Número de columnas visibles.


<select> y <option>
La etiqueta <select> se utiliza para crear un menú desplegable, y cada opción dentro del menú se define con <option>.


<button>
Se utiliza para crear un botón dentro de un formulario. Puede ser de tipo submit, reset, o button.


Validación de Formularios
Se realiza en el navegador antes de enviar los datos al servidor, utilizando atributos HTML como required, pattern, minlength, maxlength, entre otros.
